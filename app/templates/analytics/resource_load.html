Пояснение показателей i

Базовые термины

  • Занято (busy) — время из бронирований: сумма пересечений записей с выбранным периодом (и со слотом в графиках). Для нескольких сотрудников в одном слоте минуты суммируются — это суммарная потребность в ресурсе.
  • Мощность (capacity) — сколько минут работы доступно по правилам: для сотрудников — график цикла (например 2/2), смена, обед из перерывов, выходные СЦ; для ворк-зон — упрощённо по рабочему окну дня и выходным СЦ.
  • Дефицит в слоте — max(0, занято − мощность). Если мощность 0, а брони есть, дефицит покажет перегруз или несогласованность данных.

Фильтры формы

  • СЦ — если не выбран, считаются все центры из справочника.
  • Период дат — только для блоков ниже формы (KPI, график, heatmap), не для «Прогноза по механикам».
  • Ресурсы — все / только сотрудники / только ворк-зоны.
  • Категория — механики / мойщики (по типу ресурса в справочнике) или все.

Карточки KPI (основной отчёт)

  • Всего ресурсов — число строк в выборке (сотрудники и/или ворк-зоны по фильтру).
  • Задействовано / С бронированиями — сколько ресурсов имели хотя бы одну запись; в процентах — доля от «всего ресурсов».
  • Бронирований всего — число уникальных бронирований (appointment_uid), попавших в текущий фильтр ресурсов и период.
  • Средняя длительность бронирования ресурса, мин — сумма длительностей этих уникальных бронирований (в пересечении с периодом) / бронирований всего.
  • Занято часов (сумма) — сумма busy по выбранным ресурсам за период, в часах.
  • Доступно часов за период — сумма мощности по тем же ресурсам (смены 2/2, обеды, без «лишних» календарных дней без графика).
  • Утилизация за период — занято / доступно по сумме минут; главный показатель заполненности пула ресурсов.
  • Средняя загрузка — среднее арифметическое процентов загрузки по строкам ресурсов (каждый ресурс весит одинаково — может отличаться от утилизации).
  • Средняя загрузка (только занятые) — средний % только среди ресурсов с busy > 0.
  • Пиковая загрузка — максимум отношения busy/capacity в одном временном слоте (по сумме по ресурсам в слоте).
  • Дефицит часов — сумма дефицитов по всем слотам периода, в часах.
  • Слотов с дефицитом — доля слотов, где занято > мощности.
  • Окон ≥2 ч / ≥4 ч подряд — сколько раз на сетке шага «Шаг» можно начать запись такой длины: подряд идут все нужные интервалы (2 ч или 4 ч), и в каждом запас ≥ длины шага (слот целиком свободен в модели суммарной мощности). Окна внутри дня считаются без пересечений (одна запись не «перекрывает» другую), отдельно по каждому ресурсу (сотруднику/ворк-зоне), затем суммируются.
  • Средняя загрузка сотрудников / ворк-зон — отдельно по типу ресурса, без учёта второго типа.
  • Сотрудники: механики / мойщики; Ворк-зоны: … — средняя загрузка внутри категории (механика / мойка и т.д.).

График и таблица «Спрос vs мощность (по дням)»

  • По каждому календарному дню суммируются занято и мощность по всем слотам дня; столбики — часы в день; утилизация в таблице — отношение сумм за день.

Heatmap дефицита

  • Ячейка «день недели × час» — средний дефицит в минутах по таким слотам за выбранный период; 0 — нехватки не было.

Прогноз по механикам (от сегодня)

  • Не зависит от дат в форме. Окна: с сегодняшней даты до +3 и до +7 календарных дней включительно.
  • Только сотрудники-категория «механик»; СЦ — как в фильтре (или все, если СЦ не выбран).
  • Утилизация / Занято / Доступно — те же правила, что и в основном отчёте, но на фиксированном горизонте.
  • Бронирований — число записей с механиком, попавших в окно.
  • Окон ≥2 ч / ≥4 ч подряд / Дефицит часов — как в KPI, но только на этом окне.

Рекомендация по штату

  • Ориентировочная оценка по суммарному дефициту и длине периода (FTE); для решений смотрите утилизацию и прогноз.
